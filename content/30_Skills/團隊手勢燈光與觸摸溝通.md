# 團隊手勢、燈光與觸摸溝通 (Team Hand, Light & Touch Communication)

側掛常見於低能見度、頂部封閉或雙手被佔用（捲線器、手電筒）的環境，溝通系統因此分為三層：**手勢 → 燈光 → 觸摸**，依能見度與距離遞補。溝通的鐵律是：**每個信號都需要回應確認**——傳達不算完成，對方理解才算完成 [1][2]。

---

## ✋ 技術潛水手勢慣例

### 與休閒手勢的差異
*   技潛手勢以**單手**為主——另一手常握著手電筒或捲線器 [2]。
*   數字（報氣壓用）採單手五指制：1–5 直伸手指、6–9 為橫向手指組合、0 為 OK 圈。報 "150 bar" = 比 1、5、0。
*   戴三指手套（乾衣冷水）時手勢需簡化調整，團隊應在潛水前簡報中統一 [4]。

### 核心手勢（洞穴/技潛通用）[1]
| 手勢 | 意義 | 備註 |
| :--- | :--- | :--- |
| OK 圈 | 詢問/回報正常 | **必須回應** |
| 握拳 | Hold—停住等待 | 全隊停止移動 |
| 拇指向上 | 中止潛水/上升 | **不可討價還價**，任何人比出即全隊執行 |
| 食指+拇指手槍狀 | Out & Up—立即出洞/撤退 | 緊急信號 [1] |
| 雙指8字繞 | 纏線警告 | 提示隊友線已纏繞 [1] |
| 手刀切喉 | 沒氣 (OOG) | 接長管捐氣程序（見 [[氣源耗盡 (OOG) 與長管分享程序]]） |

## 🔦 燈光信號 (Light Signals)

沿導引線前進時隊形前後排列、手勢看不到，燈光是主要溝通工具 [1]：

| 光束動作 | 意義 |
| :--- | :--- |
| 在隊友視野內的表面**畫小圈** | OK？/ OK。（需回圈確認） [1] |
| **緩慢左右掃動** | 注意——非緊急，請看我 [1] |
| **快速大幅甩動** | **緊急**——立即支援（如 OOG） [1] |

*   規矩：**光圈打在隊友看得到的地面/岩壁上**，不是照隊友的臉（會毀掉對方暗適應）。
*   隊形中的燈光紀律：各自光束保持在自己前方掃視區，光束亂晃本身就是「有狀況」的被動信號——隊友燈光異常靜止或亂動都應主動查看。

## 🤝 觸摸溝通 (Touch Contact)

能見度歸零（揚沙、湧泥、燈全滅）時的最後手段。雙方以**握線手保持環扣在導引線上**，另一手接觸隊友手臂傳令 [1]：

| 觸摸信號 | 意義 |
| :--- | :--- |
| 用力捏一下 | 停 (Stop) [1] |
| 向後拉 | 後退 [1] |
| 向前推 | 前進 [1] |

*   零能見度下的隊形與出洞程序詳見 [[頂部封閉環境下的側掛線路與操作]] 與 [[失散與迷失應變程序]]。

## 🧭 隊形與溝通節奏

1.  **潛前簡報必含信號複習**：全隊確認每個信號的意義一致，特別是跨機構潛伴（PADI/GUE/洞穴體系手勢略有差異）[1][2]。
2.  **定期氣壓回報**：領隊以「?」+ 指壓力錶詢問，隊員單手報數；側掛報**較低那瓶**或依隊規報雙瓶。
3.  **並排 vs 前後**：開放水域並排（手勢為主）；線上前後（燈光為主）。轉換隊形時領隊以燈光信號告知。
4.  **潛後檢討 (Debrief)**：溝通不順的信號當天檢討修正，是團隊默契累積的核心 [1]。

2. **Scuba Diver Life** - *Communicating While Technical Diving*: 技潛單手信號的理由（另一手被佔用）、單手數字報氣壓、三指手套下的手勢調整。 [連結](https://scubadiverlife.com/communicating-technical-diving/)
3. **GUE (Global Underwater Explorers)** - *Underwater Communication*（官方教材 PDF，付費）: GUE 體系從 Open Water 到 Tech 的標準化手勢與燈光信號手冊。 [連結](https://www.gue.com/store/underwater-communication-pdf)
4. **Divernet** - *Hand Signals in Diving: Bridging the Communication Gap Underwater*: 技潛單手五指信號的跨機構通用性與戴厚手套/三指手套時的手勢調整。 [連結](https://divernet.com/scuba-diving/hand-signals-in-diving-bridging-the-communication-gap-underwater/)
